Step1

POWERON

  • DHCP

DHCPD returns network info, next-server, and PXE file-name

Step2

BOOT

  • TFTP

TFTP request for pxelinux.0, kernel, and boot source. We also deliver a url to ksconfig

Step3

Kickstart

  • HTTP
  • XFF
  • Packages
  • Partitioning

The host requests a ksconfig file from a badejump url. The host's MAC address is read from the X-Forwarded-For header and used to retrieve data and render the ksconfig file, which is then returned to the host

Step4

Config MGMT

  • Chef
  • Puppit
  • CFEngine

When the host is ready (Kickstart-POST), we'll drop an agent in place to take the base OS and provision it into its final state
